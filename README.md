# 彩数实验室 · LottoLab

本地运行的彩票数据工作台，支持**双色球**与**大乐透**。真实开奖数据 + 两个 AI（Claude / Codex）各自独立的确定性推算模型 + 完整对奖核算。

## 必须先说清楚的事

彩票每期开奖**相互独立、完全随机**，不存在任何能提高中奖概率的算法——包括本项目里的两套 AI 模型。它们做的是"基于真实历史数据的确定性结构化推算"：同样的数据永远算出同样的号，方法透明、可复现、可审计，但**不是预测**。对奖与统计功能是精确的，选号功能是娱乐。

## 功能

- **开奖大厅**：最新一期 + 近 20 期真实开奖（含奖池/公告奖级），开奖倒计时，任意期搜索（期号/日期，全量档案），点击任意一期展开奖级公告明细
- **数据分析**：号码频率、当前遗漏热力、奇偶形态分布、和值走势、奖池走势；统计窗口可切换「近 150 期 / 全量历史」
- **数据推理**：双脑主推一注（hero 位，可一键存入票夹）+ 共识号码池 + 可展开的双脑十组明细（Claude Fable 5 × Codex），近150期/全量双窗口切换；下方预测台账·开奖对照·双窗记分牌
- **机选工具**：娱乐性机选——完全随机 / 热号加权 / 冷号回补 / 形态均衡；可带入对奖或存入票夹
- **实证回测**：全量数据滚动回测两套模型 vs 纯随机 vs 理论值（含 95% 置信区间）、蒙特卡洛验证器、已证伪陈列室（两件展品：模拟取最高频 / 误差反馈自我优化）、数据溯源与可复现性清单
- **对奖中心**：单式/复式自动识别，组合计数公式精确对奖；**一、二等浮动奖自动按当期开奖公告实发奖金计**（无公告期回落估值）；「我的票夹」记录真实购票，开奖后自动对奖、累计真实盈亏（localStorage 本机存储）

## 准确率与信服度（先读这一段）

回测结论（300 期 × 5 注，滚动窗口无未来函数）：**两套 AI 模型 ≈ 纯随机 ≈ 理论值**，95% 置信区间均覆盖理论期望。这不是模型失败，而是彩票的数学本性——开奖独立随机，历史数据不含未来信息。本平台的"准确率"指**校准**（标注概率=实际频率），"信服度"指**可验证性**（数据可溯源、算法零随机数可复现、回测协议公开、阴性结果原样展示）。窗口大小也不影响精度：50/150/500/1000/3000 期窗口实测命中率全部 ≈ 理论值，区别只在选号"个性"强弱。

## 双脑模型

| | Claude 模型（Fable 5 设计） | Codex 模型（OpenAI Codex 设计） |
|---|---|---|
| 文件 | `scripts/fable_model.py` | `scripts/codex_model.py` |
| 思路 | 自适应噪声检验收缩（卡方比较冷热偏离与纯随机噪声，收缩力度由数据自定）+ 形态分层组合设计（5 注按历史奇偶×和值分布构成"分层代表团"） | 位置形态分布 + 号码共现图 + 相邻期转移矩阵，全枚举 110 万组合打分（查表加速） |
| 共性 | 确定性、零随机数、同输入同输出 | 同左 |

LLM 只在**设计期**出场（各自编写算法脚本），**运行期没有任何模型 API 调用**——这是结果可复现、可审计、离线可用的原因。Claude 侧已历三代：v1 `claude_model.py`（频率/热度/遗漏）→ v2 `opus_model.py`（Opus 4.8 设计，Dirichlet 后验+信息熵形态）→ v3 `fable_model.py`（现役）；退役版本均保留作对照，升级流程=新模型重写算法→确定性校验→全量回测对照（CI 覆盖理论值）→上线。共识池 = 两套模型选号的交集，仅表示两种统计视角重合，不代表概率优势。

## 使用

**推荐：双击 `启动彩数实验室.command`**（首次运行如被 macOS 拦截，右键 → 打开）。它会启动本地服务并自动打开浏览器，页面右上角出现「↻ 更新数据 · 双脑重算」按钮——点一下即可在页面内完成 拉取真实开奖 → Claude 模型 → Codex 模型 → 合并共识 全流程，实时显示日志，完成后自动刷新。建议每个开奖日后点一次。

也可以手动启动：`python3 server.py`（默认 127.0.0.1:8765，被占用时自动顺延）。

兼容模式：直接双击 `index.html`（file:// 打开）功能照常，只是没有页内刷新按钮，此时用终端刷新：

```bash
./scripts/ai_pick.sh
```

等价于依次执行：`fetch_data.py`（拉真实开奖）→ `claude_model.py` → `codex_model.py` → `merge_picks.py`。

## 迁移到新机器

整个平台的数据都在项目文件夹内（台账 `data/ledger.js`、票夹 `data/wallet.json`、全量档案等），迁移三步：

1. 把整个 `lottolab` 文件夹拷到新机器（AirDrop / 移动硬盘 / scp 均可）
2. 新机器上跑 `./scripts/install_autostart.sh` 重装定时任务（脚本自动适配新路径）
3. 双击 `启动彩数实验室.command` 即可——浏览器票夹数据与 `data/wallet.json` 双向同步，老机器浏览器里的票会在拷贝的 wallet.json 里

依赖只有 macOS 自带的 python3（标准库，零 pip 依赖）；codex/claude CLI 仅在想重新设计模型时才需要，日常运行不依赖。票夹页有「导出备份」按钮可随时手动备份。

## 数据源

- 双色球：福彩官网 API 优先，境外出口 IP 被拦时自动落到 500.com 行情站
- 大乐透：体彩官网 API 优先，同上自动回落
- 各保留最近 150 期，落地为 `data/draws.js`
